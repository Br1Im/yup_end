# YUP — план развития продукта

Дорожная карта от MVP до PMF, с конкретикой по стеку, фичам, рискам и срокам. Под соло-фаундера + RU-первичный рынок.

---

## Стратегические ставки (4 решения, на которых строится всё)

### Ставка 1 — Telegram Mini App = основная платформа
**Не делай нативные мобильные приложения. Не делай десктоп. Не делай PWA отдельно.**

Причины:
- Apple Pay не работает в РФ. Google Play частично. **Telegram Stars работают с любого устройства.**
- Для русских студентов 17–20 Telegram = их основной мессенджер. **Friction регистрации = 0.**
- Web push на iOS — боль. **Telegram-уведомления работают везде.**
- Telegram Mini App = просто веб (HTML/JS/CSS), запущенный внутри ТГ. Один кодбейс.
- Bot API даёт тебе бесплатный канал «дотянуться до пользователя» — push-уведомления, daily-чек-ины, всё нативно.

**Результат:** один продукт, один кодбейс, нулевая friction регистрации, нативные платежи через Telegram Stars.

### Ставка 2 — лендинг + Mini App, **никакого отдельного веб-приложения**
Лендинг (тот, что есть) — публичная вершина воронки. Все CTA ведут на Telegram-бота, который открывает Mini App. Нет отдельной web-версии приложения. Один путь — через Telegram.

Это **сильно** упрощает стек, маркетинг и аналитику.

### Ставка 3 — AI = OpenAI gpt-4o-mini, не GPT-4
Реалистичная экономика:
- gpt-4o-mini: $0.15 / 1M input tokens, $0.60 / 1M output tokens
- Один пользователь = ~10–15 AI-вызовов в месяц (генерация плана + ежедневный re-routing + еженедельная корректировка)
- Средний расход на пользователя: **~$0.10–0.20/мес**
- При премиуме 199–299₽/мес ($2–3) — маржа 90%+

Не используй GPT-4 в проде. gpt-4o-mini хватает с запасом для этой задачи. **Качество плана зависит от prompt engineering**, не от размера модели.

### Ставка 4 — pre-built modules > custom content
**Не пиши свой контент.** Не пиши свои уроки. AI собирает план из:
- Best public courses (YouTube, Stepik, Coursera)
- Книги (с указанием конкретных глав)
- Apps (рекомендация Anki / Drops для слов)
- Workout templates (на тренировки)

Ты строишь **навигатор**, не **школу**. Это критическое позиционирование. Если пытаешься делать свой контент — конкурируешь со Skyeng / Stepik / GeekBrains, и проигрываешь.

## MVP — что ВНУТРИ, что СНАРУЖИ

### Внутри MVP (ship за 4–6 недель)

**Core loop:**
1. **Onboarding** (3 экрана):
   - Описать цель свободным текстом («хочу выучить английский до B2 за год»)
   - 2–3 уточняющих вопроса (текущий уровень, время в день, deadline)
   - AI генерирует план на 30 дней
2. **Главный экран = «Сегодня»:**
   - Список из 1–3 шагов на сегодня
   - Каждый шаг — конкретный (не «учи английский 30 минут», а «посмотри видео X на YouTube + сделай 10 карточек в Anki»)
   - Кнопка «Сделал» / «Не успел» / «Частично»
3. **Re-routing при пропуске:**
   - Пропустил 1 день — план продолжается с того же шага
   - Пропустил 3+ дней — AI пересоберёт план, спросит что произошло
4. **Прогресс-экран:**
   - Текущая позиция на маршруте (например, «День 12 / 30»)
   - Простая визуализация (полоса или восхождение)
5. **Один пользователь = одна цель** (на free tier).
6. **Telegram-уведомления** утром: «Сегодня 3 шага, 25 минут».

### Снаружи MVP (отложить до v2+)

- Несколько целей одновременно — премиум фича
- AI-коуч с диалогом (просто чат с ботом) — v2
- Стрики, бейджи, achievements — v2 (retention layer)
- Социалка, лидерборды, друзья — v3
- Аналитика прогресса (графики, метрики) — v2
- Экспорт в календарь — v3
- Команды / групповые цели — v4
- Свой контент-движок — никогда (см. ставку 4)
- Нативные мобильные приложения — никогда до 5k MRR

**Принцип:** MVP делает **одну вещь идеально** — превращает «хочу X» в «сегодня я делаю Y». Всё остальное — после первых платящих.

## Фазы развития

### Фаза 0 — фундамент (недели 1–2)

**Цель:** настроить инфраструктуру так, чтобы фаза 1 могла идти быстро.

- [ ] Создать Telegram-бота (@yup_app_bot), зарегистрировать Mini App URL
- [ ] Backend skeleton: FastAPI / Express / NestJS — что ближе. Минимально: один эндпоинт `/api/init` для проверки initData от Telegram.
- [ ] DB schema (Postgres):
  ```
  users(id, tg_id, locale, created_at, premium_until)
  goals(id, user_id, title, raw_description, created_at)
  routes(id, goal_id, ai_plan_json, day_count, created_at)
  daily_steps(id, route_id, day_number, steps_json, status, completed_at)
  ai_calls(id, user_id, prompt_tokens, completion_tokens, cost, purpose, ts)
  ```
- [ ] Подключить OpenAI gpt-4o-mini, написать **first prompt-template** для генерации плана.
- [ ] Подключить Sentry или PostHog с самого начала. Без аналитики ты летишь вслепую.
- [ ] Mini App — пустой Next.js, инициализация TG WebApp SDK, тёмная тема.

**Deliverable к концу недели 2:** пользователь открывает бот, нажимает «Начать», видит экран «Опиши цель», описывает, видит «AI готовит план...» — даже если план пока заглушка. End-to-end проводка.

### Фаза 1 — рабочий MVP-loop (недели 3–6)

**Цель:** одна цель, AI генерирует план, daily check-in работает.

- [ ] Onboarding flow: текстовое описание + 2–3 follow-up вопроса (выбор из options)
- [ ] AI plan generation: prompt template, парсинг ответа в structured JSON, сохранение в `routes`
- [ ] Today screen: показывает шаги дня, кнопки check-in
- [ ] Re-routing logic: при пропуске 3+ дней — пересборка
- [ ] Progress screen: простой бар «День X / Y»
- [ ] TG-notifications: cron-job утром для всех активных пользователей
- [ ] Soft-launch на 30 человек: близкий круг + первые 20 из waitlist

**Deliverable:** работающий end-to-end MVP. Цель → план → ежедневный шаг → re-routing.

### Фаза 2 — retention layer (недели 7–10)

К этому моменту у тебя есть данные: где пользователи бросают, какие планы плохие, какие шаги работают.

- [ ] **Стрики** — мягкие, как у Duolingo (1 пропуск не сжигает). Главный retention-driver у этой ЦА.
- [ ] **AI-fix loop:** если 3 шага подряд оценены как «слишком сложно» — AI облегчает план; «слишком легко» — усложняет.
- [ ] **Weekly recap-уведомление:** «За неделю — 5/7 дней, прогресс 18%».
- [ ] **«Re-engagement»:** если пользователь не заходил 3+ дня — автоматическое сообщение от бота с предложением вернуться (pause / continue / restart).
- [ ] **Onboarding A/B:** 2 версии онбординга → какая даёт больше completion rate.
- [ ] **Качество плана:** prompt-engineering на основе реальных данных. Ловишь моменты «AI выдал хрень», добавляешь в prompt few-shot examples.

**Deliverable:** D14 retention 25%+, D30 retention 12%+. Без этого — нет смысла идти в фазу 3.

### Фаза 3 — монетизация + multi-goal (недели 11–14)

К этому моменту у тебя 50–200 активных пользователей и понимание, что работает.

- [ ] **Telegram Stars** интеграция: подписка через нативный TG-checkout
- [ ] **Premium tier:** 199–299₽/мес или 990–1490₽/полугодие через Stars
- [ ] **Multi-goal:** до 3 целей одновременно (premium-only). AI учитывает общую загрузку.
- [ ] **AI-coach (диалог):** чат с ботом, можно задавать вопросы по плану. Premium-only.
- [ ] **Реферальная программа:** 3 друга → месяц премиума бесплатно
- [ ] **Web-версия Mini App** для не-русских пользователей (если решишь активировать EN)

**Deliverable:** первые 30+ платящих. MRR 10k–20k₽. PMF-сигнал #1.

### Фаза 4 — глубина и удержание (месяцы 4–6)

Здесь ты уже не строишь, а оптимизируешь.

- [ ] **Длинный режим:** план на 3–6 месяцев вперёд, не только на 30 дней
- [ ] **Cross-domain insights:** «ты лучше учишь язык в дни, когда тренируешься» — уникальная фича, которой нет у конкурентов
- [ ] **AI-rituals:** утренний короткий разговор с ботом «как ты сегодня» → план дня учитывает состояние
- [ ] **Сообщество:** общий TG-чат для платящих, weekly «Camp Day»
- [ ] **Voice-первый интерфейс:** надиктовать цель / отчёт голосом → AI разбирает (Telegram это нативно поддерживает)
- [ ] **Интеграции:** Anki, Notion, Google Calendar (export-only)

**Deliverable:** 100+ платящих, MRR 30k–50k₽, D90 retention 8%+, NPS 40+. Это сигнал, что можно открывать EN-канал.

## Стек (рекомендуемый)

| Слой | Что | Почему |
|---|---|---|
| **Лендинг** | Next.js + Vercel (текущий) | Уже есть, статика, бесплатно |
| **Mini App frontend** | Next.js + telegram-web-app SDK + Tailwind | Можно использовать тот же подход что и лендинг — реюз компонентов |
| **Backend API** | FastAPI (Python) или NestJS (Node) | FastAPI — быстро, type-safe, native AI streaming. NestJS — если ты лучше знаешь TS |
| **DB** | Postgres + Supabase / Neon | Бесплатный tier на старте, до 1k пользователей не платишь |
| **Auth** | Telegram WebApp initData валидация на бэке | Никаких форм входа |
| **AI** | OpenAI gpt-4o-mini через official SDK | Стабильно, дёшево, хватает |
| **Платежи** | Telegram Stars (RU) + Stripe (EN, потом) | Stars нативно, Stripe для EN |
| **Hosting backend** | Fly.io / Railway / Render | Бесплатные tier-ы для старта |
| **Cron** | Fly Machines / Vercel Cron | Для daily notifications |
| **Аналитика** | PostHog (self-hosted или free tier) | Funnel, retention, A/B — всё в одном |
| **Логи / errors** | Sentry | Бесплатный tier хватает |

**Не делай:**
- Микросервисы (один backend monolith — норм до 10k пользователей)
- Кубернетес (избыточно)
- Свой ML-pipeline (используй OpenAI)
- GraphQL (избыточно для CRUD)
- Redis (Postgres + jsonb хватает на старте)

## Критические риски и mitigation

### Риск 1 — AI генерит generic / плохие планы
**Симптом:** пользователи говорят «ну, план как из ChatGPT, ничего особенного».
**Mitigation:**
- Few-shot examples в prompt — 3–5 примеров «отличного плана» для разных целей
- Структурированный output (JSON schema), не свободный текст — заставляет AI давать конкретику
- Включай в prompt контекст пользователя (возраст, уровень, расписание, предыдущие результаты)
- Быстрая итерация: каждую неделю смотришь 10 планов, улучшаешь prompt

### Риск 2 — пользователь не возвращается на 2-й день
**Симптом:** D1 retention < 50%.
**Mitigation:**
- TG-notification утром — единственный способ удержания на старте
- Первый день — ОЧЕНЬ маленькие шаги (5–10 минут максимум). Дать ощущение «я смог» в первый день.
- Не показывать сразу всю длину пути (30 дней страшно). Показывать только сегодня + завтра.

### Риск 3 — стоимость AI растёт быстрее выручки
**Симптом:** при росте до 500 пользователей AI ест $50–100/мес, а платящих 10.
**Mitigation:**
- Ограничь AI-вызовы для free tier (1 генерация плана / 1 еженедельный re-route)
- Кэшируй похожие планы — если 100 пользователей попросили «английский на B2», базовый шаблон одинаковый, AI только адаптирует
- Перейди на gpt-4o-mini-batch если нужно (50% дешевле, асинхронно)

### Риск 4 — Telegram заблокируют в РФ или поменяют правила Mini App
**Mitigation:** держи Mini App как просто веб-app, чтобы при необходимости отвязать от TG за 1 неделю. Фактически — у тебя должен быть веб-URL, который открывается и без TG (просто без auth). Это бесплатная страховка.

### Риск 5 — ты выгораешь
**Симптом:** код-стек MVP занял 12 недель вместо 6, и ты устал.
**Mitigation:**
- Ставь жёсткий scope. **Каждую фичу из «снаружи MVP» сначала отвергай**, потом доказывай зачем.
- Минимум 1 день в неделю без кода (только маркетинг / общение с пользователями)
- Не пиши идеальный код. Пиши тот, что работает. Рефакторишь после PMF.

## TL;DR

1. **Платформа:** Telegram Mini App + лендинг (тот, что есть). Никаких нативных приложений, никакой отдельной web-версии.
2. **Стек:** Next.js + FastAPI / NestJS + Postgres (Supabase) + OpenAI gpt-4o-mini + Telegram Stars.
3. **MVP за 4–6 недель:** одна цель, AI-план на 30 дней, daily check-in, re-routing при пропуске. Всё.
4. **Фазы:** фундамент (нед. 1–2) → MVP loop (3–6) → retention layer (7–10) → монетизация (11–14) → глубина (мес. 4–6).
5. **Главные ставки:** TG Mini App, gpt-4o-mini (не GPT-4), Telegram Stars, никакого своего контента.
6. **Главные риски:** generic AI-планы, плохой D1, AI-cost при росте, выгорание. Mitigation у каждого есть.
7. **Главный принцип:** ship to 5 real users in 14 days. Не «идеальный продукт перед запуском», а 5 человек, которые вчера попробовали.
